## 5. 重建最容易踩的雷

1. **無後端、無資料庫**：本系統完全在瀏覽器執行，所有資料存於 localStorage。不需要啟動任何伺服器或資料庫。

2. **直接開啟 HTML 會報 CORS 錯誤（開發模式）**：開發時用 `npm run dev`，不要直接開啟 `index.html`。打包後的 `index.html`（`npm run build`）才可以直接用瀏覽器開啟。

3. **`vite-plugin-singlefile` 是關鍵依賴**：若沒有這個外掛，打包後的 HTML 會有 `type="module"` 的 script，在 `file://` 協議下因 CORS 無法載入。確保 `package.json` 中有此依賴。

4. **密碼懶遷移**：`SEED_USERS` 中的密碼是明文（`Admin@2024!`），首次登入後會自動 SHA-256 雜湊並更新 localStorage。若清除 localStorage，再次登入時重新觸發遷移，屬正常行為。

5. **班表 dateKey 格式不補零**：儲存 key 為 `"2026-7-13"` 而非 `"2026-07-13"`。若手動操作 localStorage 要注意此格式。

6. **localStorage 容量限制約 5MB**：員工人數多或資料量大時可能達到上限。系統有配額警告機制，但不會自動壓縮資料。

7. **vite.config.js 的 outDir 指向上層目錄**：建構輸出至 `../委外人力排班作業平台/index.html`，確保此資料夾存在，否則建構會失敗。
