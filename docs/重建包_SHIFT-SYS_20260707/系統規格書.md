## 1. 系統概覽

| 項目 | 內容 |
|------|------|
| 系統名稱 | 委外人力排班作業平台 |
| 系統代號 | SHIFT-SYS |
| 系統用途 | 管理物流倉儲委外人力的排班、休假與班別設定，支援多角色存取控制 |
| 主要使用者 | 系統管理員、當區幹部、委外廠商幹部 |
| 系統版本 | v1.0.0 |
| 規格書版本 | v1.0 |
| 建立日期 | 2026/07/07 |

### 核心目標

> 提供物流倉儲業務的委外人力班表管理工具，讓當區幹部能統一管理各廠商員工的排班作業，並讓委外廠商幹部能在權限範圍內自行調整所屬員工班表。系統採純前端架構（無後端伺服器），所有資料儲存於使用者瀏覽器的 localStorage，打包成單一 HTML 檔案，方便在無網路環境下使用。

## 3. 資料模型

> ⚠️ 無資料庫 schema。所有資料以 JSON 格式儲存於 localStorage，key 前綴為 `sms_`。

### 3.1 localStorage 資料表一覽

| localStorage Key | 資料結構 | 說明 |
|-----------------|---------|------|
| `sms_users` | `User[]` | 所有帳號（含角色、權限） |
| `sms_employees` | `Employee[]` | 員工清冊 |
| `sms_schedule` | `{ [empId]: { [dateKey]: code } }` | 班表（日期鍵格式：`YYYY-M-D`） |
| `sms_shift_types` | `ShiftType[]` | 班別設定（日/中/夜班定義） |
| `sms_shiftcode_rows` | `ShiftCodeRow[]` | 班別代號表資料列 |
| `sms_shiftcode_headers` | `string[]` | 班別代號表欄位標題 |
| `sms_year` | `number` | 目前選擇的年份 |
| `sms_month` | `number` | 目前選擇的月份 |
| `sms_schedule_range` | `{ start, end }` | 排班區間設定（可跨月） |
| `sms_open_holidays` | `string[]` | 啟用的國定假日清單 |
| `sms_warehouses` | `Warehouse[]` | 倉庫/部門/組別設定 |
| `sms_system_locked` | `boolean` | 系統鎖定狀態 |

### 3.2 重要資料結構詳細說明

#### User（帳號）
```javascript
{
  id: string,           // 唯一 ID（如 "u0"）
  username: string,     // 登入帳號
  password: string,     // sha256:... 格式（懶遷移：舊版明文，登入時自動升級）
  role: 'ADMIN' | 'AREA' | 'VENDOR',
  name: string,         // 顯示名稱
  vendors: string[],    // 可管理的廠商清單（VENDOR 角色使用）
  system: boolean,      // 系統預設帳號（不可刪除）
  allowedWarehouses: string[],
  approved: boolean,
  loginCount: number,
  permissions: object,
}
```

#### Employee（員工）
```javascript
{
  id: string,           // 內部唯一 ID
  empId: string,        // 員工編號（如 "CY11206238"）
  name: string,
  vendor: string,       // 所屬廠商名稱
  warehouse: string,    // 所屬倉庫
  dept: string,         // 部門
  group: string,        // 組別
  shiftTypeId: string,  // 指定班別 ID（對應 ShiftType.id）
}
```

#### 班表 dateKey 格式
- 格式：`"YYYY-M-D"`（例：`"2026-7-13"`，月日不補零）
- 班表值：`'V'`（上班）、`'休'`（休假）、`'國'`（國定假日）、`''`（空）
- 遷移邏輯：舊版以純數字（`"13"`）為 key，系統啟動時自動轉換

### 3.3 班表 key 遷移

啟動時若 localStorage 中的 schedule 使用舊格式（數字 key），App 的 `useState` 初始化函式會自動遷移至完整日期 key，結合當時儲存的年月組合。

## 4. API 端點

> 本系統無後端 API。所有資料操作均透過 React `Context + setState` + localStorage 完成。

### 4.1 資料存取模式

| 操作 | 實作方式 |
|------|---------|
| 讀取資料 | `LS.get(key, fallback)` — 從 localStorage 讀取並 JSON 解析 |
| 寫入資料 | `LS.set(key, value, onQuotaError)` — JSON 序列化後寫入 localStorage |
| 全域狀態 | React Context (`AppContext`)，包含所有主要 state 與 setter |
| 配額錯誤 | `LS.set` 第三參數為錯誤回呼，觸發時顯示 Toast 警告 |

### 4.2 認證機制

- 登入：在 `users` 陣列中比對 username + 驗證 SHA-256 密碼
- 無 Token / Session，登入狀態僅存於 React state（重新整理需重新登入）
- 密碼格式：`sha256:<hex>`；舊版明文密碼於首次登入時自動升級為雜湊

## 5. 商業邏輯

### 5.1 排班區間（rangeMode）

- 系統設定中可設定排班區間（`scheduleRange.start` ~ `scheduleRange.end`）
- 設定後，班表管理改為顯示指定日期區間，不再以月份為單位
- 支援跨月顯示（如 7/13 ~ 8/9）
- `viewOffset`：整數，每次 ±1 使顯示區間平移一個週期長度，實現「上/下個週期」瀏覽
- 班表儲存 key 使用完整 `YYYY-M-D` 格式，確保跨月資料不互相覆蓋

### 5.2 班別輪值週期（SHIFT_CYCLE）

```
V（上班） → 休（休假） → 國（國定假日） → ''（空） → V ...
```

點擊班表格子循環切換。VENDOR 角色不可設定「國」；「休」受每月配額限制。

### 5.3 休假配額控制

- 每位員工每月 `休` 的天數有上限（由系統設定的班別定義）
- 廠商幹部排假時，若超出配額則顯示警告
- 當月已用配額在排班時即時計算

### 5.4 連續上班警示

- **條件**：員工在可見日期範圍內連續 ≥ 6 天 `V`（上班）
- **顯示**：連續的格子底色改為粉紅色（`bg-pink-200`）
- **適用角色**：管理員與當區幹部；廠商登入時不顯示

### 5.5 7日斑馬紋週期

- 班表欄位每 7 天為一組，奇偶組以不同底色區分（`bg-slate-500` vs 預設）
- 週六日文字以黃色顯示
- 跨月份時在月份起始欄位顯示藍色左邊框與月份標籤

### 5.6 班別代號轉換

- 每位員工設有 `shiftTypeId`，對應班別代號表（sms_shiftcode_rows）
- 「顯示代號」模式下，V/休/國 等原始碼依代號表轉換為班別專屬代號
- 匯出「代碼轉換」Excel 時使用轉換後的代號

### 5.7 員工清冊匯入規則

- 以員工編號（empId）為唯一識別
- 重複匯入時：更新基本資料（姓名、廠商、倉庫等），但**保留**既有的 `shiftTypeId`
- 尚未設定班別的員工優先排列在清冊上方
- 每次匯入限制最多 1000 筆（防止 localStorage 超限）

### 5.8 Excel 班表匯入

支援兩種格式自動偵測：

| 格式 | 識別方式 | 員編欄 | 日期格式 | 資料起始列 |
|------|---------|--------|---------|-----------|
| 系統匯出格式 | 第1欄標題為「員工編號」 | 第1欄（col 0） | `7/13(一)` | 第2列（row 1） |
| 外部 Excel 格式 | 第1欄標題為「廠商」 | 第3欄（col 2） | `7/13` | 第3列（row 2） |

值對應：`V`→上班、`休`/`(休)`→休假、`國`→國定假日、數字→上班（V）、`-`/空→跳過

### 5.9 國定假日

- 系統內建 `NATIONAL_HOLIDAYS` 陣列（可在系統設定啟用/停用）
- 啟用的假日在班表對應日期自動標記為「國」

### 5.10 計算公式

| 指標 | 公式 | 說明 |
|------|------|------|
| 出勤天數 | `count(V)` in 可見日期範圍 | 不含休、國、空 |
| 休假使用量 | `count(休)` in 當月 | 用於配額判斷 |
| 連續上班判斷 | 連續 `V` 天數 ≥ 6 | 觸發粉底警示 |

## 6. 存取控制

### 6.1 角色說明

| 角色代碼 | 名稱 | 說明 | 資料存取範圍 |
|---------|------|------|------------|

| `AREA` | 當區幹部 | 管理所轄倉庫的排班與人員 | 指定倉庫 |
| `VENDOR` | 委外廠商幹部 | 管理所屬廠商員工的班表 | 指定廠商員工（寫入限制） |

| 人員清冊（查看/匯入） | ✅ | ✅ | ✅（限所屬廠商） |
| 人員班別設定 | ✅ | ✅ | ❌ |
| 報表匯出 | ✅ | ✅ | ✅（限所屬廠商） |
| 班別代號表 | ✅ | ✅ | ❌ |
| 系統設定 | ✅ | ❌ | ❌ |

| 權限管理 | ✅ | ❌ | ❌ |
| 連續上班粉底警示 | ✅ | ✅ | ❌ |

### 6.3 存取控制實作

- **登入入口區分**：員工入口（ADMIN/AREA）、廠商入口（VENDOR）
- **VENDOR 寫入限制**：修改班表前檢查 `emp.vendor` 是否在 `currentUser.vendors` 內，否則拒絕並顯示錯誤
- **上方篩選器**：倉庫/部門/組別篩選同時套用於班表管理與人員清冊

## 8. 部署架構

### 8.1 部署方式

產品為**單一自含 HTML 檔案**，包含所有 JS、CSS、字型（inline 或 base64）。
可透過以下方式使用：
- 直接雙擊開啟（`file://`）
- 放置於任何靜態 Web 伺服器
- 上傳至 GitHub Pages（公開）或內部伺服器

### 8.2 建構指令

```bash
# 開發模式
npm run dev

# 產品建構（輸出至 ../委外人力排班作業平台/index.html）
npm run build
```

vite.config.js 的 `outDir` 預設輸出至 `../委外人力排班作業平台/`。

### 8.3 建構特性

- `vite-plugin-singlefile`：所有資源 inline，無外部請求
- 建構後約 1.7 MB 單一 HTML
- 不需要 `type="module"`，可在 `file://` 協議下正常運作（解決 ES module CORS 限制）

## 9. 資料保護說明

### 9.1 個人資料

| 欄位 | 所在資料 | 資料內容 | 儲存方式 |
|------|---------|---------|---------|
| `name` | Employee / User | 員工姓名、帳號顯示名稱 | 明文，localStorage |
| `empId` | Employee | 員工編號 | 明文，localStorage |

### 9.2 機敏資料

| 欄位 | 所在資料 | 資料內容 | 類別 |
|------|---------|---------|------|
| schedule 整體 | sms_schedule | 員工出勤狀況（人資類） | 人資 |
| `vendor` | Employee | 廠商歸屬（人力配置） | 人資 |

### 9.3 資料保護措施

| 措施 | 說明 |
|------|------|
| 密碼儲存 | SHA-256（Web Crypto API），格式 `sha256:<hex>`；舊版明文於首次登入時懶遷移升級 |
| 傳輸加密 | 無後端傳輸；若部署至 HTTPS 靜態伺服器則由伺服器保障 |
| 資料隔離 | 資料僅存於本機 localStorage，不上傳至任何伺服器 |
| XSS 防護 | PDF 匯出時所有使用者輸入資料經 `esc()` HTML escape 處理 |
| VENDOR 寫入隔離 | VENDOR 角色僅能修改所屬廠商員工資料，程式碼層面強制驗證 |
