## 1. 系統概覽

| 項目 | 內容 |
|------|------|
| 系統名稱 | 委外人力排班作業平台 |
| 系統代號 | VENDOR-SHIFT |
| 系統用途 | 委外人員排班作業 |
| 主要使用者 | 系統管理員、當區幹部、各委外廠商幹部 |
| 系統版本 | v1.0.0 |
| 規格書版本 | v1.0 |
| 建立日期 | 2026/07/10 |

### 核心目標

管理多家委外廠商（如承杺、芊通、三彥等）在各倉別（大溪倉、大肚倉、岡山倉）的人員每日排班狀況。系統採「單一 HTML 檔案」打包方式，無需後端伺服器，所有資料存於瀏覽器 localStorage，便於在無網路環境下操作。管理員可設定排班鎖定期間、開放/關閉廠商幹部排國定假日權限，並提供儀表板即時掌握各廠出勤概況。

## 3. 資料模型

> ⚠️ 所有資料存於 localStorage，無後端資料庫。以下為各 key 的資料結構說明。

### 3.1 localStorage Key 一覽

| Key | 說明 |
|-----|------|
| `sms_users` | 使用者帳號列表 |
| `sms_employees` | 員工清冊 |
| `sms_schedule` | 排班資料（`{ [empId]: { [dateKey]: code } }`） |
| `sms_vendors` | 廠商清單 |
| `sms_warehouses` | 倉別/課別/組別樹狀結構 |
| `sms_shift_types` | 班別設定（上班時間） |
| `sms_shift_code_rows` | 班別代號表（上傳的 Excel 轉換後） |
| `sms_system_locked` | 排班鎖定旗標（boolean） |
| `sms_schedule_range` | 排班可編輯區間 `{ start, end }` |
| `sms_open_holidays` | 管理員開放的國定假日 key 陣列 |
| `sms_vendor_hol_open` | 委外幹部可否排「國」（boolean） |
| `sms_current_user` | 目前登入使用者（session） |

### 3.2 重要資料結構詳細說明

#### 使用者（User）

```js
{
  id: 'u0',
  username: 'admin',
  password: 'Admin@2024!',   // 明文（無後端，依賴 localStorage 隔離）
  role: 'admin',             // 'admin' | 'area' | 'vendor'
  name: '系統管理員',
  vendors: ['承杺', '芊通', ...],  // 可管轄的廠商
  allowedWarehouses: [],     // 限制可見倉別（空陣列 = 全部）
  approved: true,
  loginCount: 0,
  system: true,              // true = 系統帳號，不可停用/刪除
  permissions: { ... }       // 細粒度頁面/功能權限
}
```

#### 員工（Employee）

```js
{
  id: 'e1',
  empId: 'CS001',      // 員工編號（清冊格式：廠商代碼+序號）
  name: '員工姓名',
  vendor: '承杺',      // 所屬廠商名稱
  dept: '大肚理貨課',  // 課別
  group: '日班-理貨組', // 組別
  status: '在職',       // '在職' | '離職'
  shiftTypeId: 'st1',  // 對應班別設定 id
}
```

#### 排班（Schedule）

```js
{
  "e1": {
    "2026-7-10": "V",   // V=上班, 例=例休, 休=休假, 國=國定假日, ''=空白
    "2026-7-11": "例",
    ...
  },
  ...
}
```

dateKey 格式：`YYYY-M-D`（月、日不補零）

#### 倉別結構（Warehouse）

```js
{
  id: 'wh2',
  name: '大肚倉',
  departments: [
    {
      id: 'dept_wh2_1',
      code: 'L035',
      name: '大肚理貨課',
      vendors: [],       // 此課別綁定的廠商
      groups: ['日班-理貨組', '中班-理貨組', ...]
    }
  ]
}
```

## 5. 商業邏輯

### 5.1 班別代碼循環

點擊班表格子，依序循環：`V（上班）→ 國（國定假日）→ 例（例休）→ 休（休假）→ V`

若系統設定「委外幹部排班權限」關閉（`sms_vendor_hol_open = false`），委外幹部（role: vendor）點擊時會自動跳過「國」。

### 5.2 委外幹部每週例休限制

委外幹部（role: vendor）每週（週日～週六）最多排 **1 天例休（例）**。若本週已有 1 天「例」，再次嘗試排「例」時系統會彈出錯誤提示並取消操作。

### 5.3 班別代號轉換

系統維護一份「班別代號表」（由管理員匯入 Excel）。當員工有設定班別時，班表可顯示轉換後的代號（如 AH7、ZC8），而非原始 V/例/休/國。

轉換規則（`getDisplayCode` 函式）：
- `V` → 代號表第 1 欄（上班代號）
- `例` 或 `休` → 代號表第 2 欄（休假代號）
- `國` → 依假日名稱對應代號表欄位（精確比對當天假日）；找不到時找同月最近假日 fallback

### 5.4 連續上班警示

若同一員工連續上班 ≥ 6 天，班表格子顯示橘色警示樣式。

### 5.5 國定假日排班開放控制

管理員可在「系統設定」逐一開放/關閉特定國定假日（`sms_open_holidays`），控制哪些日期允許排「國」；另有全域開關（`sms_vendor_hol_open`）控制委外幹部是否可見「國」選項。

### 5.6 儀表板統計

- **前周差**：選定日期排班人數 vs. 7 天前同日排班人數之差值
- **各組別細分**：依員工 `group` 欄位關鍵字比對（日班理貨組、中班理貨組、日班出貨組、運務組）
- **廠商排班比例圓形圖**：以 SVG 甜甜圈圖呈現各廠商占比

## 6. 存取控制

### 6.1 角色說明

| 角色代碼 | 名稱 | 說明 | 資料存取範圍 |
|---------|------|------|------------|
| `admin` | 系統管理員 | 最高權限，可存取所有功能 | 全部廠商、全部倉別 |
| `area` | 當區幹部 | 可查看/編輯班表、管理人員，不可系統設定 | 所屬廠商、允許倉別 |
| `vendor` | 委外幹部 | 僅能編輯自己廠商的班表（限例休） | 所屬廠商 |

### 6.2 功能與角色對照

| 功能 | admin | area | vendor |
|------|:---:|:---:|:---:|
| 儀表板 | ✅ | ✅ | ✅ |
| 班表管理（查看） | ✅ | ✅ | ✅ |
| 班表管理（編輯） | ✅ | ✅ | ✅（限自廠） |
| 人員清冊（查看） | ✅ | ✅ | ✅ |
| 人員清冊（新增/編輯） | ✅ | ✅ | ❌ |
| 人員清冊（刪除/清除） | ✅ | ❌ | ❌ |
| 報表匯出 | ✅ | ✅ | ❌ |
| 班別代號表 | ✅ | ✅ | ✅（唯讀） |
| 系統設定 | ✅ | ❌ | ❌ |
| 帳號管理 | ✅ | ❌ | ❌ |

### 6.3 存取控制實作

存取控制在前端 React Context（`AppContext`）中實作。登入後 `currentUser` 存入 `sms_current_user`。頁面切換時依 `currentUser.permissions[pageKey].view` 決定是否顯示選單項目及頁面內容。

使用 React Context（`AppContext`）集中管理所有全域狀態。所有 state 於初始化時從 localStorage 讀取，每次變更後透過 `useEffect` 同步寫回。

### 7.3 與後端溝通

無後端，不適用。

## 8. 部署架構

### 8.1 部署方式

執行 `npm run build` 後，`dist/index.html` 為完整的單一 HTML 檔案（所有 JS/CSS 內嵌），可直接複製給使用者在瀏覽器開啟，或部署至任何靜態網頁伺服器。

### 8.2 啟動指令

```bash
# 開發環境
cd 排班作業
npm install
npm run dev        # http://localhost:5173

# 生產打包（輸出 dist/index.html）
npm run build
```

### 8.3 資料庫初始化

無資料庫。首次開啟瀏覽器時，若 localStorage 無資料，系統自動載入 `SEED_USERS`、`SEED_EMPLOYEES`、`SEED_VENDORS`、`SEED_WAREHOUSES` 種子資料。
