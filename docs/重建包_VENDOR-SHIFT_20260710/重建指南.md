## 2. 怎麼用這個包來重建

### 核心原則：原始碼為準

「系統規格書」負責讓你**理解**系統（架構意圖、商業規則、為什麼這樣設計）；
但**精確的欄位名稱、型別、前端邏輯，一律以 `原始碼/ShiftSystem.jsx` 為準**。
兩者衝突時，相信原始碼。

### 建議的閱讀／執行順序

1. 先讀「**系統規格書.md**」第 1～2 章，建立整體輪廓。
2. 讀規格書第 3 章（資料模型）——了解 localStorage 各 key 的結構。
3. 讀規格書第 5 章（商業邏輯）——這是系統的靈魂，務必細看。
4. 依「系統規格書」**附錄 B：重建執行手冊** 的步驟，安裝、啟動、驗收。

## 5. 重建最容易踩的雷

1. **vite-plugin-singlefile 是關鍵**：若遺漏此 plugin，`npm run build` 輸出的 HTML 會有外部 JS/CSS 連結，無法單檔離線使用。確認 `vite.config.js` 有 `viteSingleFile()`。

2. **localStorage 資料殘留**：若之前有舊版資料在 localStorage，重建後可能讀到格式不符的舊資料。首次驗收前，開啟瀏覽器 DevTools → Application → Clear Storage，清除所有 localStorage 後重整頁面。

3. **dateKey 格式不補零**：排班 key 格式為 `YYYY-M-D`（月、日不補零，如 `2026-7-5`），匯入外部資料時若補零（`2026-07-05`）將導致資料對不上。

4. **班別代號表需另行匯入**：系統本身不內建代號表資料，需由管理員在「班別代號表」頁面上傳 `shift_codes_data.json` 對應格式的 Excel，否則代碼轉換功能無效（班表仍可正常使用 V/例/休/國）。

5. **⚠️ 密碼明文儲存**：所有帳號密碼以明文存於 localStorage，部署後請立即登入「帳號管理」修改所有預設密碼（特別是 `admin / Admin@2024!`）。
